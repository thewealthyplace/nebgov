# Security Incident Response Runbook

This document outlines the procedures for responding to security alerts generated by the NebGov automated monitoring system.

## Alert Types and Response Procedures

### 1. LARGE_TRANSFER (Severity: MEDIUM/HIGH)
**Description**: Detected a significant movement of voting power (delegation change) exceeding the configured threshold.

- [ ] **Verify Source**: Identify the delegator address. Is it a known entity or a protocol-affiliated address?
- [ ] **Analyze Impact**: Calculate the new voting power percentage of the delegatee. Does this enable them to unilaterally pass or block proposals?
- [ ] **Monitor Proposals**: Check if the delegatee has created any new proposals immediately following the transfer.
- [ ] **Action**: If the transfer seems malicious or coordinated for a governance attack, contact the protocol guardians to discuss potential vetoes or emergency pauses.

### 2. PAUSE_DETECTED (Severity: CRITICAL)
**Description**: The Governor contract has been paused.

- [ ] **Identify Pauser**: Check which address triggered the pause. Was it an authorized emergency action?
- [ ] **Check Rationale**: If the pause was triggered via governance, review the corresponding proposal.
- [ ] **Assess Risks**: While paused, no new proposals can be created. Determine if any time-sensitive operations are blocked.
- [ ] **Action**: If unauthorized, investigate the compromise of the pauser address.

### 3. SUSPICIOUS_PROPOSAL (Severity: HIGH)
**Description**: A proposal was created with abnormally large calldata or suspicious target addresses.

- [ ] **Audit Targets**: Inspect the contract addresses being called. Are they known protocol contracts or external/unverified addresses?
- [ ] **Decode Calldata**: Use the SDK to decode the function arguments. Is the proposal attempting to drain the treasury or change critical settings?
- [ ] **Check Proposer**: Review the history of the proposer.
- [ ] **Action**: If malicious, use the Guardian's veto power to cancel the proposal before it reaches the voting stage.

### 4. RATE_LIMIT_HIT (Severity: MEDIUM)
**Description**: An address is attempting to create proposals faster than the allowed rate.

- [ ] **Identify Address**: Is this a bot or a spam attack?
- [ ] **Check Content**: Are the proposals identical or nonsensical (spam)?
- [ ] **Action**: No immediate action required as the contract enforces the limit, but monitor for broader Sybil attacks.
